\# 弭 -Yuhazu-



<img src="./screenshots/main-img.png" alt="弭 -Yuhazu- 動作デモ" width="800"/>



\*\*Webページ上のテキストを、ローカルPCで起動した音声合成ソフト（VOICEVOX, Aivis Speech）と連携して読み上げるChrome拡張機能です。\*\* AIチャット（ChatGPTなど）の応答を主な対象としつつ、Web小説の読み上げにもわずかながら対応しています。



\## 主な機能



\- \*\*対応音声合成ソフト\*\*

&nbsp;   

&nbsp;   \*   \[VOICEVOX](https://voicevox.hiroshiba.jp/)

&nbsp;	    

&nbsp;   \*   \[Aivis Speach](https://aivis-project.com/)    

&nbsp;       

\- \*\*自動読み上げ

&nbsp;   

&nbsp;   - ヘッダーの「AUTO」ボタンを有効にすると、ページコンテンツの更新を検知し、テキストの追記が完了したタイミングで自動的に読み上げを開始します。

&nbsp;       

\- \*\*ストリーミングモード ⚡\*\*

&nbsp;   

&nbsp;   - 自動読み上げのオプション機能です。AIチャットのように逐次生成される文章を、差分を検知して滑らかに読み上げます。（試験的機能）

&nbsp;       

\- \*\*モード切替\*\*

&nbsp;   

&nbsp;   - \*\*詳細モード:\*\* 話者設定やテキスト取得設定など、全ての機能にアクセスできる標準のUIです。

&nbsp;       

&nbsp;   - \*\*シンプルモード:\*\* ヘッダーの⚙ボタンから、再生・停止など基本操作のみに絞ったコンパクトなUIに切り替えられます。

&nbsp;       

\- \*\*ノイズ対策\*\*

&nbsp;   

&nbsp;   - 読み上げ時にノイズになりやすい文末の 」 ） といった不要な記号を、読み上げ対象から除外するよう設定できます。

&nbsp;       

\- \*\*話者分離\*\*

&nbsp;   

&nbsp;   - アリス：「こんにちは」 のように、特定のトリガー文字列に応じて話者を自動で切り替えることができます。

&nbsp;       

\- \*\*クリック位置からの再生\*\*

&nbsp;   

&nbsp;   - UI上のテキストエリア内のお好みの場所をクリックすると、その位置から読み上げを開始できます。

&nbsp;       



\## インストール方法



1\. \*\*リリースページからZIPファイルをダウンロード\*\*

&nbsp;   

&nbsp;   - 本リポジトリの最新リリースページから Source code (zip) をダウンロードします。

&nbsp;       

2\. \*\*ZIPファイルを解凍\*\*

&nbsp;   

&nbsp;   - ダウンロードしたZIPファイルを任意の場所に解凍（展開）します。

&nbsp;       

3\. \*\*Chromeで拡張機能ページを開く\*\*

&nbsp;   

&nbsp;   - Chromeのアドレスバーに chrome://extensions と入力して、拡張機能管理ページを開きます。

&nbsp;       

4\. \*\*デベロッパーモードを有効化\*\*

&nbsp;   

&nbsp;   - ページの右上にある「デベロッパーモード」のトグルスイッチをONにします。

&nbsp;       

5\. \*\*拡張機能を読み込む\*\*

&nbsp;   

&nbsp;   - 左上に表示される「パッケージ化されていない拡張機能を読み込む」ボタンをクリックします。

&nbsp;       

&nbsp;   - ファイル選択ダイアログが開くので、ステップ2で解凍した\*\*フォルダそのもの\*\*を選択します。

&nbsp;       

&nbsp;   - 「弭 -Yuhazu-」が一覧に表示されればインストール完了です。

&nbsp;       



\## 使い方



1\. \*\*音声合成ソフトを起動する\*\*

&nbsp;   

&nbsp;   - PC上で対応する音声合成ソフトのいずれか、または両方を起動しておきます。

&nbsp;       

2\. \*\*UIを表示する\*\*

&nbsp;   

&nbsp;   - 読み上げたいWebページを開き、ブラウザのツールバーにある「弭 -Yuhazu-」のアイコンをクリックしてUIを起動します。

&nbsp;       

3\. \*\*話者リストを更新する\*\*

&nbsp;   

&nbsp;   - 「話者設定」ボタンを押し、ポップアップ内の「話者リストを更新」ボタンをクリックします。起動中のソフトから話者情報が自動で取得されます。

&nbsp;       

4\. \*\*読み上げ対象を選択する\*\*

&nbsp;   

&nbsp;   - 「テキスト取得設定」を開き、「取得対象」のプルダウンから、閲覧しているサイトに合ったものを選択します。（例: ChatGPT）

&nbsp;       

5\. \*\*テキストを取得して再生\*\*

&nbsp;   

&nbsp;   - 「テキストを取得」ボタンを押すと、指定された部分の文章がテキストエリアに表示されます。

&nbsp;       

&nbsp;   - 再生ボタン（▶）を押すと、読み上げが開始されます。

&nbsp;

\#### 自動再生モードについて



\##### 1. モードの有効化・無効化



\- \*\*有効化\*\*:

&nbsp;   

&nbsp;   - UI上部にある「AUTO」ボタン（またはシンプルモードの「AUTO」ボタン）を押すと有効になります。

&nbsp;       

&nbsp;   - ボタンが緑色に点灯し、Webページ内のテキスト監視（MutationObserver）が開始されます。

&nbsp;       

&nbsp;   - ストリーミングモードの稲妻（⚡）ボタンが操作可能になります。

&nbsp;       

\- \*\*無効化\*\*:

&nbsp;   

&nbsp;   - 有効化されている「AUTO」ボタンを再度押すと無効になります。

&nbsp;       

&nbsp;   - テキストの監視が停止します。

&nbsp;       

&nbsp;   - \*\*手動で「停止（■）」ボタンを押した場合も、自動再生モードは自動的に無効化されます。\*\*

&nbsp;       



\##### 2. 読み上げが実行される仕組み



1\. \*\*テキストの監視\*\*:

&nbsp;   

&nbsp;   - 自動再生モードが有効になると、「テキスト取得設定」で指定されたCSSセレクタに一致する要素の監視を開始します。

&nbsp;       

&nbsp;   - 例えば、ChatGPTの返信部分（div.markdowntなど）が監視対象となります。

&nbsp;       

2\. \*\*変更の検知\*\*:

&nbsp;   

&nbsp;   - 監視対象の要素内で、テキストの追加や変更が検知されると、タイマーが作動します。

&nbsp;       

3\. \*\*読み上げの実行\*\*:

&nbsp;   

&nbsp;   - タイマーが作動してから一定時間、追加の変更がなければ「変更が完了した」と判断し、読み上げ処理を開始します。

&nbsp;      



\##### 3. 2つのモードの違い



\*\*A) 通常の自動再生モード\*\* (AUTOボタンのみ有効)



\- \*\*目的\*\*: テキスト全体が完全に表示されてから読み上げることを想定しています。

&nbsp;   

\- \*\*待ち時間\*\*: テキストの変更が止まってから \*\*1.5秒後\*\* に読み上げを開始します。

&nbsp;   

\- \*\*動作\*\*:

&nbsp;   

&nbsp;   1. AIの返信などでテキストが変化し始めると、内部でタイマーを開始します。

&nbsp;       

&nbsp;   2. テキストの変更が続く間はタイマーをリセットし続けます。

&nbsp;       

&nbsp;   3. 変更が1.5秒間止まると、「返信が完了した」とみなし、その時点での\*\*全てのテキスト\*\*を読み上げます。

&nbsp;       

\- \*\*利点\*\*: 一文ずつ途切れ途切れになることなく、まとまった文章として自然に聞くことができます。

&nbsp;   

\- \*\*欠点\*\*: 長文の返信の場合、全てが表示されるまで待つ必要があります。

&nbsp;   



\*\*B) ストリーミングモード\*\* (AUTOと⚡ボタンの両方が有効)



\- \*\*目的\*\*: AIがテキストを生成している最中から、追従するようにリアルタイムで読み上げることを想定しています。

&nbsp;   

\- \*\*待ち時間\*\*: テキストが変更されてから \*\*0.1秒〜3.0秒後\*\* （スライダーで調整可能、デフォルトは0.3秒）に読み上げを開始します。

&nbsp;   

\- \*\*動作\*\*:

&nbsp;   

&nbsp;   1. AIの返信などで少しでもテキストが追加されると、短い待ち時間（例: 0.3秒）で即座に読み上げを開始します。

&nbsp;       

&nbsp;   2. 前回読み上げた部分からの\*\*差分（追加された部分）\*\*を特定し、その部分のみを読み上げます。

&nbsp;	  

\- \*\*利点\*\*: 応答が速く、すぐに内容を聞き始めることができます。

&nbsp;   

\- \*\*欠点\*\*: 文章が不自然に途切れたり、同じ部分を繰り返してしまったりする可能性があります。

&nbsp;   



\#### 話者分離のルールについて



\- トリガー文字列が検出されてから、次のトリガーまたは文章の終わりまでが、指定した話者の読み上げ対象です。

&nbsp;   

\- 同じ行の中にトリガー文字列が複数含まれる場合は、より後ろにあるものが優先されます。

&nbsp;   

\#### テキスト取得設定の手動登録



Webページの構成が変わりテキストを取得できなくなった際に、ご自身で対象を指定できます。



1\. 取得したいテキストの上で右クリックし、メニューから「検証」を選択してブラウザのDevToolsを開きます。

&nbsp;   

2\. DevTools上で、取得したいテキストがハイライトされる要素のclass名などを参考に、CSSセレクタを特定します。

&nbsp;   

3\. 拡張機能の「テキスト取得設定」で、特定したCSSセレクタと任意の名前を登録します。

&nbsp;   

このページの場合は　div.flex.justify-start　が該当します。

\### ファイル構成



\- manifest.json: 拡張機能の名前、バージョン、権限などを定義する設定ファイルです。

&nbsp;   

\- background.js: 音声合成エンジンとの通信や設定の保存など、目に見えない裏側の処理を担当します。

&nbsp;   

\- content\_style.css: ページ上に表示される操作パネルやポップアップの見た目を整えるスタイルシートです。

&nbsp;   

\- content\_ui.js: UIのHTMLを生成し、ボタンクリックなどのユーザー操作を検知します。

&nbsp;   

\- content.js: background.js と content\_ui.js をつなぎ、拡張機能全体の動作ロジックを統括します。

&nbsp;   

\- defaults.js: 拡張機能を初めて使う際の、話者やCSSセレクタなどの初期設定データです。
